🚀 Installation Guide

FormFlow AI – AI‑Powered Form Builder SaaS

Thank you for purchasing FormFlow AI! This guide will walk you through the installation process step by step. The application is built on Laravel 12 and requires a standard web server with PHP and MySQL.

⚡ Quick Tip: If you have vendor/ folder included in the package, you can skip composer install and jump to step 4.

📋 Requirements

  • PHP 8.2 or higher (with extensions: BCMath, Ctype, Fileinfo, JSON, Mbstring, OpenSSL, PDO, Tokenizer, XML, GD)
  • MySQL 8.0 or higher (or MariaDB 10.5+)
  • Composer (if you need to install dependencies)
  • Node.js (only if you want to rebuild frontend assets with Vite)
  • Web Server (Apache / Nginx with PHP-FPM)

📦 Installation Steps

  1. Upload files
    Upload the entire contents of the FormFlowAI/ folder to your web server's document root (e.g., public_html or /var/www/html).
  2. Set permissions
    Make sure the following directories are writable by your web server: 
    chmod -R 755 storage bootstrap/cache
chmod -R 755 public/uploads   (if the folder exists)
    On shared hosting, you can do this via cPanel File Manager (right‑click → Change Permissions).
  3. Install dependencies (if vendor folder is missing)
    Run Composer to install all PHP dependencies: 
    composer install --optimize-autoloader --no-dev
    If you have the vendor/ folder already, you can skip this step.
  4. Create environment file
    Copy .env.example to .env: 
    cp .env.example .env
    Or rename it manually via file manager.
  5. Generate application key
    This sets the encryption key for your application: 
    php artisan key:generate
  6. Configure database
    Open .env in a text editor and update these values: 
    DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=your_database_name
DB_USERNAME=your_db_username
DB_PASSWORD=your_db_password
    Also set your Gemini API key, mail settings, and optional Stripe/Twilio keys if you plan to use those features.
  7. Run migrations & seeders
    This creates all tables and populates the default templates and subscription plans: 
    php artisan migrate --seed
  8. Create storage link
    This allows public access to uploaded files (e.g., file uploads in forms): 
    php artisan storage:link
  9. Build frontend assets (optional)
    If you want to rebuild the CSS/JS assets using Vite: 
    npm install
npm run build
    If you don't need to modify the frontend, you can skip this – the pre‑built assets are already in the public/ folder.
  10. Set up cron job (for scheduled tasks)
    Add this to your server's crontab: 
    * * * * * php /path/to/your/project/artisan schedule:run >> /dev/null 2>&1
    This is required for any scheduled notifications or background jobs.
  11. Configure web server
    Point your domain or subdomain to the public/ directory.
    • Apache: The .htaccess file inside public/ handles routing automatically.
    • Nginx: Use this location block: 
      location / {
    try_files $uri $uri/ /index.php?$query_string;
}
  12. Verify installation
    Visit your site's URL in a browser. You should see the splash page with the “Login” and “Sign Up” buttons.
    • Register a new user → you'll be taken to the dashboard.
    • Start creating forms and exploring features.

🛠️ Troubleshooting

  • “No application encryption key” error: Run php artisan key:generate.
  • “403 Forbidden” on public files: Check folder permissions; storage/ must be writable.
  • Database connection error: Double‑check your .env database credentials.
  • 404 on routes (Nginx): Ensure your server is configured to pass requests to index.php.
  • AI Generator not working: Verify your GEMINI_API_KEY in .env is valid.
  • Mail not sending: Check SMTP settings in .env and ensure your mail provider allows external SMTP.
🔐 Security Note: For production, set APP_ENV=production and APP_DEBUG=false in your .env file. Always keep your .env file outside of public access.
